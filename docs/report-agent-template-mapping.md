샘플 보고서를 기준으로 보고서는 아래 11개 표와 종합의견 본문 문단을 채우는 구조입니다.

## Sections

1. 기본 정보
2. 집행 내역 요약
3. 증빙 유효성 검증 결과
4. 세금 및 정산 결과
5. 항목별 적정성 검토 결과
6. 부적정 및 검토 필요 상세 내역
7. 보완 필요 사항
8. 종합 의견

## Renderer Notes

표 8-10은 샘플에 기본 3개 블록으로 들어있지만, 실제 데이터에서는 이슈 개수가 달라질 수 있습니다. `renderer.py`는 이슈 개수에 맞춰 6번 상세 블록을 조정합니다.

- 이슈가 1~3건이면 불필요한 기본 블록을 제거합니다.
- 이슈가 4건 이상이면 마지막 상세 표 스타일을 복제해 `6.4`, `6.5` 순서로 보완 필요 사항 섹션 앞에 추가합니다.
- 이슈가 없으면 빈 상세 블록 1개만 남깁니다.

종합 의견은 별도 표가 아니라 `8. 종합 의견` 아래 본문 문단에 반영합니다.
