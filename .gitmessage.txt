# type(scope): subject  ← 이 줄을 지우고 커밋 메시지를 작성하세요 (50자 이내, 마침표 없음)
#
# body (선택) ← 빈 줄 한 줄 후 본문 작성 (무엇을, 왜 변경했는지)
#
#
# ──────────────────────────────────────────────
# 🏷️  타입 (type)
# ──────────────────────────────────────────────
#   feat     : 새로운 기능 추가
#   fix      : 버그 수정
#   docs     : 문서 변경 (README, 주석 등)
#   style    : 코드 의미 변화 없는 포맷 수정
#   refactor : 기능 변화 없는 코드 구조 개선
#   test     : 테스트 코드 추가 및 수정
#   perf     : 성능 개선
#   chore    : 빌드, 패키지 설정 등 기타 변경
#
# ──────────────────────────────────────────────
# ✅  커밋 메시지 규칙
# ──────────────────────────────────────────────
#   - 제목은 50자 이내, 마침표 없음
#   - 제목은 명령형으로 작성  예) "수정함" ❌  "수정" ✅
#   - 제목과 본문 사이 빈 줄 한 줄
#   - 본문은 무엇을, 왜 변경했는지 설명 (어떻게는 코드로)
#   - 한 커밋에 하나의 논리적 변경만 포함
#   - 영문 작성 시 첫 글자는 소문자
#
# ──────────────────────────────────────────────
# 💡  좋은 예시
# ──────────────────────────────────────────────
#   feat(auth): 소셜 로그인 기능 추가
#   fix(api): 회원가입 시 이메일 중복 검사 오류 수정
#   docs(readme): 설치 방법 업데이트
#   refactor(user): 유저 서비스 레이어 분리
#
# ──────────────────────────────────────────────
# 🌿  현재 브랜치 네이밍 규칙
# ──────────────────────────────────────────────
#   feature/login-page          새 기능 (develop 분기)
#   fix/signup-validation       버그 수정
#   hotfix/payment-crash        긴급 수정 (main 분기)
#   release/v1.2.0              배포 준비
#   · 소문자 + 하이픈(-)만 사용, 이슈 번호 포함 권장
